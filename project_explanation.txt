## Project Overview

Your project is a deep learning application that identifies plant diseases from leaf photos. Think of it as a "doctor" for plants — you give it a photo of a sick leaf, and it tells you what disease it has, how confident it is, and how severe the disease appears. Here's everything broken down:

### DATA FOLDER — Handling Images

[data/dataset.py]
- What it does: Loads and prepares plant leaf images for training
- Key features:
  - TARGET_CLASSES: Defines the 10 disease types the model recognizes:
    - Tomato: Early Blight, Late Blight, Healthy (3)
    - Potato: Early Blight, Late Blight, Healthy (3)
    - Corn: Common Rust, Healthy (2)
    - Apple: Apple Scab, Healthy (2)
    - Image preprocessing

  - Images are converted to a fixed size : Neural networks require same image size.
  - Image augmentation: Makes training images slightly different each time (rotate, blur, change brightness) so the model learns to recognize diseases under various real-world conditions (smartphone angles, lighting, etc.)
  - Train/Val/Test split: Divides data into 70% for training, 15% for validation (tuning), 15% for testing
  - Normalization: Adjusts image pixel values to be in a standard range so the model learns better

[data/download_dataset.py]
- What it does: Automatically downloads the PlantVillage dataset from GitHub (~800 MB)
- Key features:
  - Downloads from official GitHub repository
  - Extracts only the 10 disease classes you need
  - Organizes images into folders by disease type
  - Shows progress and per-class image counts

### MODELS FOLDER — The Neural Network

[models/model.py]
- What it does: Defines the neural network architecture (the "brain" that makes predictions)
- Key concepts:
  - Transfer Learning: Uses EfficientNet-B0 (by default), a pre-trained model that already knows how to recognize features in images from being trained on millions of pictures (ImageNet)
  - Why? Training from scratch would take forever; starting with knowledge saves time and improves accuracy
  - Classifier Head: A small layer added on top to specifically classify your 10 disease types
  - Two operational modes:
    - Frozen backbone: Keep the pre-trained knowledge fixed, only train the disease classifier
    - Unfrozen backbone: Fine-tune the entire network for better performance
  - Helper methods:
    - freeze_backbone() — locks pre-trained layers during Phase 1
    - unfreeze_backbone() — allows last 30% of layers to adapt during Phase 2
    - get_target_layer() — identifies where to extract visualization data for explanations

### TRAINING FOLDER — Teaching the Model

[training/train.py]
- What it does: The main script that teaches the model to classify diseases
- Two-phase training strategy:
  - Phase 1 (Frozen backbone, ~10 epochs):
    - Pre-trained layers stay locked
    - Only the classifier head (new disease recognition layer) is trained
    - Learning rate: Standard (1e-4)
    - Goal: Quickly adapt pre-trained features to your disease types
  - Phase 2 (Fine-tuning, ~10 epochs):
    - Last 30% of backbone layers unlocked
    - Everything adjusts carefully at a lower learning rate (0.1x)
    - Goal: Fine-tune all features for your specific task
- Early Stopping:
  - If validation loss stops improving for 5 epochs, training stops early to prevent overfitting (memorizing training data instead of learning patterns)
- Metrics tracked:
  - Training/validation loss (error)
  - Training/validation accuracy (correctness %)
- Outputs:
  - Best checkpoint saved to models/best_model.pth
  - Class names saved to models/class_names.json
  - Training curves plot showing how accuracy improved

### EVALUATION FOLDER — Testing the Model

[evaluation/evaluate.py]
- What it does: Tests the trained model on unseen data to measure performance
- Metrics generated:
  - Accuracy: Overall correctness percentage
  - Precision/Recall/F1 per class: Detailed per-disease performance
    - Precision: Of predictions marked as "disease X", how many were correct?
    - Recall: Of all actual "disease X" cases, how many did we catch?
  - Confusion Matrix: Visual heatmap showing which diseases the model confuses (e.g., does it confuse Early Blight with Late Blight?)
- Output:
  - Classification report saved as text
  - Confusion matrix visualization saved as image

### EXPLAINABILITY FOLDER — Understanding the Model's Decisions

[explainability/gradcam.py]
- What it does: Creates visual heatmaps showing which parts of the leaf the model "looked at" to make its decision
- How it works (Grad-CAM = Gradient-weighted Class Activation Mapping):
  - Uses calculus (gradients) to trace back through the neural network
  - Identifies which pixels influenced the prediction most
  - Creates a colored overlay showing "attention regions"
  - Red areas = most important for the prediction
  - Blue areas = less important
- Why it matters: Builds trust in the model — you can see if it's looking at the right leaf symptoms

[explainability/severity.py]
- What it does: Estimates disease severity as a percentage
- How it works:
  - Uses the Grad-CAM heatmap as a proxy
  - Counts pixels where activation is high (above 0.45 threshold)
  - Calculates percentage of affected area
- Severity categories:
  - Healthy: < 5% activation
  - Mild: 5-25%
  - Moderate: 25-50%
  - Severe: > 50%
- Note: This is a rough heuristic, not pixel-perfect segmentation (would need manual annotations for that)

### INFERENCE FOLDER — Making Predictions

[inference/predict.py]
- What it does: Loads a trained model and makes predictions on new images
- Process:
  1. Loads a leaf image from disk
  2. Pre-processes it (resize to 224x224 pixels, normalize)
  3. Runs prediction through the trained model
  4. Generates Grad-CAM heatmap for explainability
  5. Estimates disease severity
  6. Returns: predicted disease, confidence %, severity %, heatmap overlay
- Output: A dictionary with prediction results

## Key Machine Learning Concepts Explained

- Transfer Learning: Using a pre-trained model as a starting point instead of training from scratch
- Backbone: The pre-trained image recognition engine (EfficientNet-B0)
- Classifier Head: A small layer that learns your specific task (disease classification)
- Augmentation: Artificially creating variations of training images (rotate, blur) to prevent overfitting
- Early Stopping: Stopping training when validation performance stops improving
- Grad-CAM: Technique to visualize which image regions the model attended to
- Confidence: The model's probability that its prediction is correct
- Precision/Recall: Two ways to measure accuracy for each disease class
